## 2.4 帧间跟踪流水线

**⏱ 2 周**

### 系统结构

这是**整个 SLAM 最核心的部分**。将所有零件组装起来：

```
每帧处理:
  1. 提取 ORB 特征
  2. 与上一帧匹配
  3. 如果第一帧：初始化 (构造初始 3D 地图)
  4. 否则：
     a. 从匹配中获取 3D-2D 对应
     b. PnP + RANSAC → 当前帧位姿
     c. 当前帧与左右图立体匹配 → 三角化新 3D 点
     d. 新 3D 点转换到世界坐标 → 加入地图
     e. 判断是否关键帧
     f. 更新状态 (prev_frame = current_frame)
```

### 🔧 实现任务

```python
# 3. 可视化相机轨迹（matplotlib）
# 4. 对比有 BA 和无 BA 的轨迹差异
```

### 验证指标

```
精度指标：
  - 绝对轨迹误差 (ATE)：与真值对比
  - 相对位姿误差 (RPE)：每米/度误差
  - 追踪成功率：成功帧数/总帧数
  
在没有真值的情况下：
  - 轨迹是否平滑（无剧烈跳变）
  - 最后是否回到起点附近（闭环检测前）
  - 重投影误差是否稳定 < 2px
```

### 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| PnP 内点少 | 误匹配多 | 提高比率测试阈值 |
| 轨迹突然跳变 | 跟踪丢失 | 检测内点数量，过低时重置 |
| 地图点不增长 | 三角化失败 | 检查视差是否足够 |
| BA 发散 | 初始值差 | 先固定一些参数或加正则化 |

# 第三阶段：后端优化

## 3.1 束调整 (BA)

**⏱ 2 周**

### 核心概念

BA = 同时优化相机位姿 + 3D 点，最小化重投影误差。

$$
\min_{R_i, t_i, X_j} \sum_{i,j} \| \pi(R_i X_j + t_i) - x_{ij} \|^2
$$

### 雅可比矩阵推导

这是 BA 中最需要理解的数学部分。

**相机位姿的雅可比（李代数扰动模型）：**

$$
\frac{\partial e}{\partial \delta\xi} = 
\begin{bmatrix}
\frac{f_x}{Z'} & 0 & -\frac{f_x X'}{Z'^2} & -\frac{f_x X'Y'}{Z'^2} & f_x + \frac{f_x X'^2}{Z'^2} & -\frac{f_x Y'}{Z'} \\
0 & \frac{f_y}{Z'} & -\frac{f_y Y'}{Z'^2} & -f_y - \frac{f_y Y'^2}{Z'^2} & \frac{f_y X'Y'}{Z'^2} & \frac{f_y X'}{Z'}
\end{bmatrix}
$$

**3D 点的雅可比：**

$$
\frac{\partial e}{\partial X} = 
\begin{bmatrix}
\frac{f_x}{Z'} & 0 & -\frac{f_x X'}{Z'^2} \\
0 & \frac{f_y}{Z'} & -\frac{f_y Y'}{Z'^2}
\end{bmatrix} \cdot R
$$

### 🔧 实现任务

```python
# homework_10_ba.py

# 1. 先实现一个小型 BA（3 相机 + 10 点）
# 2. 手工推导并实现雅可比矩阵
# 3. 用 scipy.least_squares 对比验证（自动求导）
# 4. 扩展到所有相机和点
# 5. 可视化：优化前后的重投影误差分布
```

### 稀疏性

BA 雅可比矩阵是**极度稀疏**的——每个点只出现在少数相机中。利用稀疏性可以将求解复杂度从 $O((n+m)^3)$ 降到 $O(n^3 + m)$（舒尔补）。

```
   b. 与所有关键帧的词袋向量计算相似度
   c. 相似度最高的前 N 个作为候选
   d. 几何验证：特征匹配 + 基础矩阵
   e. 时序一致性：连续多帧确认
```

$$


```cpp
    }
};
```

# 第五阶段：扩展与改进

## 5.1 改进方向

### 📈 精度提升

| 改进 | 方法 | 难度 |
|------|------|------|
| SIFT 替代 ORB | 更精确的特征 | ⭐ |
| SuperPoint + SuperGlue | 深度特征 | ⭐⭐⭐ |
| 直接法 + 特征法融合 | DSO + ORB-SLAM | ⭐⭐⭐⭐ |
| IMU 融合 | 预积分 + VIO | ⭐⭐⭐⭐ |
| 语义 SLAM | 物体级别地图 | ⭐⭐⭐⭐⭐ |

### 🚀 性能优化

| 优化 | 方法 | 效果 |
|------|------|------|
| 八叉树空间划分 | 加速特征匹配 | 2x |
| 多级网格 | 加速地图搜索 | 3x |
| GPU 特征提取 | ORB/SIFT 加速 | 5-10x |
| SSE/AVX 优化 | 矩阵运算加速 | 2-4x |

### 🌍 应用场景适配

| 场景 | 问题 | 方案 |
|------|------|------|
| 低纹理 | 特征太少 | 直接法 / 线特征 |
| 快速运动 | 跟踪丢失 | IMU / 大搜索半径 |
| 大场景 | 地图太大 | 地图拆分 / 定位模式 |
| 动态物体 | 误匹配 | 语义分割 / 运动一致性 |

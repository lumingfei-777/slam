## 5.1 针孔相机模型 🔴

### 5.1.1 最简单的相机

想象一个暗箱，前面有一个小孔，光线通过小孔在后方成像：

```
       3D 物体
          ↑
          │
  光 ────→┤ 小孔
          │
          ↓
       倒立的像
```

为了方便，我们把成像平面翻到前方（数学上等价）：

```
          P = (X, Y, Z)
              │
              │
    ┌───────┴───────┐ 相机光心 O
    └───────┬───────┘
            │
            ▼
        p = (u, v)  图像平面
```

### 5.1.2 投影公式 🔴

$$u = f_x \cdot \frac{X}{Z} + c_x$$
$$v = f_y \cdot \frac{Y}{Z} + c_y$$

```python
def world_to_pixel(X, Y, Z, fx, fy, cx, cy):
    """3D 点投影到像素坐标"""
    u = fx * X / Z + cx
    v = fy * Y / Z + cy
    return u, v
```

**💡 核心直觉：**
- 除以 $Z$：近大远小
- $f_x$ 乘：焦距越长，图像越大
- $c_x, c_y$：图像中心偏移

### 5.1.3 矩阵形式

$$s \begin{bmatrix} u \\ v \\ 1 \end{bmatrix} = 
\underbrace{\begin{bmatrix} f_x & 0 & c_x \\ 0 & f_y & c_y \\ 0 & 0 & 1 \end{bmatrix}}_{K}
\begin{bmatrix} X \\ Y \\ Z \end{bmatrix}$$

$K$ 就是**内参矩阵**。

---

## 5.2 内参矩阵 K 🔴

### 5.2.1 K 包含什么

$$K = \begin{bmatrix}
f_x & 0 & c_x \\
0 & f_y & c_y \\
0 & 0 & 1
\end{bmatrix}$$

| 参数 | 含义 | 典型值（848×480） |
|------|------|------------------|
| $f_x$ | x 方向焦距（像素） | ~596 |
| $f_y$ | y 方向焦距（像素） | ~540 |
| $c_x$ | 主点 x 坐标 | ~424 |
| $c_y$ | 主点 y 坐标 | ~240 |

### 5.2.2 💡 焦距的物理意义

```
f_x = 物理焦距(mm) × (图像宽度 / 传感器宽度)

所以 f_x 的单位是"像素"，不是毫米。
f_x 越大 → 视场角越小 → "放大倍数"越大
```

### 5.2.3 完整投影

考虑相机在世界坐标系中的位姿 $R$（旋转）和 $t$（平移）：

$$s\begin{bmatrix}u\\v\\1\end{bmatrix} = K \underbrace{\begin{bmatrix}R & t\end{bmatrix}}_{\text{外参}}
\begin{bmatrix}X\\Y\\Z\\1\end{bmatrix}$$

```python
def full_project(K, R, t, points_world):
    """完整投影：世界坐标 → 像素"""
    # 世界 → 相机
    pts_cam = (R @ points_world.T + t.reshape(3, 1)).T
    # 相机 → 像素
    u = K[0,0] * pts_cam[:,0] / pts_cam[:,2] + K[0,2]
    v = K[1,1] * pts_cam[:,1] / pts_cam[:,2] + K[1,2]
    return np.stack([u, v], axis=1)
```

## 5.3 畸变 🟡

### 5.3.1 为什么有畸变

真实相机用透镜代替小孔，透镜会引入畸变：

```
正常          桶形畸变        枕形畸变
┌────┐        ┌────┐        ┌────┐
│    │        │╭──╮│        │╲  ╱│
│    │   →    ││  ││   →    │ ╲╱ │
│    │        │╰──╯│        │ ╱╲ │
└────┘        └────┘        └────┘
```

### 5.3.2 去畸变

```python
# 用 OpenCV 去畸变
undistorted = cv2.undistort(img, K, dist_coeffs)
```

**⚠️ SLAM 中一般先对图像去畸变再处理，这样假设投影模型是线性的，简化计算。**

## 5.4 坐标系 🔴

### 5.4.1 三个坐标系

```
世界坐标 W (固定的全局坐标系)
    │
    │ 位姿 T_wc = [R | t]
    ▼
相机坐标 C (原点在相机光心)
    │
    │ 内参 K
    ▼
像素坐标 (u, v)
```

### 5.4.2 💡 SLAM 中的坐标系约定

```
相机坐标：Z 指向前方（相机看的方向）
    X 指向右
    Y 指向下（图像的行方向）

世界坐标：第一帧的相机坐标
   原点在第一帧的相机光心
```

### 5.4.3 变换链

```python
# 世界点 → 相机点
P_cam = R @ P_world + t

# 相机点 → 世界点
P_world = R.T @ (P_cam - t)

# 相机点 → 像素
u = fx * P_cam[0] / P_cam[2] + cx
v = fy * P_cam[1] / P_cam[2] + cy

# 像素 → 相机点（需要深度！）
P_cam[0] = (u - cx) * Z / fx
P_cam[1] = (v - cy) * Z / fy
P_cam[2] = Z  # Z 是深度，需要已知
```

## 5.5 对极几何 🔴

### 5.5.1 问题

同一场景的两张照片之间有什么关系？

```
左图              右图
  p₁  ← 3D点 P →  p₂
  │                 │
  │   E 或 F        │
  └──── 约束 p₂ ────┘
```

### 5.5.2 本质矩阵 E

本质矩阵 $E = t^\wedge R$ 描述了两帧之间的几何关系：

- $R$：帧间的旋转
- $t$：帧间的平移
- $t^\wedge$：$t$ 的反对称矩阵

**核心公式（对极约束）：**

$$p_2^T E p_1 = 0$$

含义：左图点 $p_1$ 在右图上的对应点 $p_2$ 一定在一条直线上（极线）。

### 5.5.3 八点法求 E

```python
# 给定两帧间的匹配点对，求本质矩阵
E, mask = cv2.findEssentialMat(pts1, pts2, K, method=cv2.RANSAC)
```

### 5.5.4 从 E 恢复 R, t

```python
_, R, t, mask = cv2.recoverPose(E, pts1, pts2, K)
```

这一步是**单目 SLAM 初始化的核心**。

## 5.6 极线搜索 🟡

### 5.6.1 什么是极线

给定左图的一个点，它在右图上的对应点一定在一条直线上——这条线就是**极线**。

```
左图点 p₁ → 在右图上画一条极线
            → p₂ 一定在这条线上
            → 只需要在这条线上搜索匹配！
```

**好处：** 将 2D 搜索降为 1D 搜索，更快、更准、更少误匹配。

### 5.6.2 在双目 SLAM 中

双目相机的左右图是**已经校正好的**（极线与图像行对齐），所以匹配只在同一行上搜索——这就是**立体匹配**的基础。

## 5.7 三角化 🔴

### 5.7.1 问题

已知匹配点 $p_1$（左图）和 $p_2$（右图），以及两帧的位姿，求 3D 点 $P$。

### 5.7.2 解法

```python
# 用 OpenCV 的 triangulatePoints
P1 = K @ np.hstack([np.eye(3), np.zeros((3, 1))])    # 左投影矩阵
P2 = K @ np.hstack([R, t.reshape(3, 1)])             # 右投影矩阵

pts_4d = cv2.triangulatePoints(P1, P2, pts_left, pts_right)
pts_3d = pts_4d[:3] / pts_4d[3]  # 齐次 → 3D
```

### 5.7.3 💡 精度分析

三角化的精度取决于**视差**（两个视角之间的夹角）：

```
视差小（远处物体）→ 深度误差大
视差大（近处物体）→ 深度误差小
基线长 → 视差大 → 深度准
```

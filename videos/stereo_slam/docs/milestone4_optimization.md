# Milestone 4：后端优化与闭环检测

## 1. 概述

最后这个阶段实现了 SLAM 的**后端优化**模块，通过**束调整（Bundle Adjustment）**和**闭环检测**来减小累积漂移，构建全局一致的轨迹和地图。

```
前端 (跟踪 + 建图)
    │
    ├── 关键帧序列
    ├── 地图点
    └── 特征匹配关系
         │
         ▼
后端优化
    │
    ├── 局部 BA (滑动窗口)
    │     └── 优化最近 N 个关键帧 + 可见点
    │
    └── 闭环检测
          └── 检测回访位置 → 全局位姿图优化
```

---

## 2. 束调整（Bundle Adjustment）

### 2.1 问题定义

BA 是 SLAM 中最核心的优化算法，它同时优化**相机位姿**和**3D 点位置**，使所有观测的**重投影误差**最小。

**目标函数：**

$$
\min_{R_i, t_i, X_j} \sum_{i=1}^m \sum_{j=1}^n \rho\left(\left\| \pi(R_i X_j + t_i) - x_{ij} \right\|^2\right)
$$

其中：
- $R_i, t_i$：第 $i$ 个相机的位姿
- $X_j$：第 $j$ 个地图点的 3D 坐标
- $x_{ij}$：点 $j$ 在相机 $i$ 上的观测像素坐标
- $\pi(\cdot)$：投影函数（内参 + 畸变）
- $\rho(\cdot)$：鲁棒核函数（抑制外点）

### 2.2 参数化

每个相机位姿有 **6 个参数**（旋转向量 3 + 平移 3），每个 3D 点有 **3 个参数**。总参数 = $6m + 3n$。

对于 3 个关键帧和 100 个可见地图点，总参数 = $18 + 300 = 318$ 个。

### 2.3 观测方程

每个 3D 点 $X$ 被相机 $[R \mid t]$ 观测到的重投影：

```python
X_c = R @ X + t                    # 转到相机坐标
u_proj = fx * X_c[0] / X_c[2] + cx # 投影到像素
v_proj = fy * X_c[1] / X_c[2] + cy
error_x = u_proj - u_observed      # 重投影误差
error_y = v_proj - v_observed
```

### 2.4 求解器

我们使用 `scipy.optimize.least_squares` 的 Trust Region Reflective（TRF）方法：

```python
result = least_squares(
    fun, x0,
    method='trf',       # Trust Region Reflective
    max_nfev=10,        # 最大迭代次数
    ftol=1e-2,          # 函数值收敛阈值
    xtol=1e-2,          # 参数收敛阈值
)
```

**为什么选择 TRF 而不是 LM？**
- LM 要求残差数 ≥ 参数数，初期可能不满足
- TRF 支持有界优化，更鲁棒
- TRF 在欠定系统中也能工作

### 2.5 局部 BA vs 全局 BA

| 类型 | 范围 | 计算量 | 效果 |
|------|------|--------|------|
| **局部 BA** | 最近 N 个关键帧 + 可见点 | 小 | 局部一致性 |
| **全局 BA** | 所有关键帧 + 所有点 | 极大 | 全局一致性 |

当前实现使用**局部 BA**（滑动窗口 $N=3$），固定最早的关键帧不变，优化窗口内其他相机的位姿和共同可见的 3D 点。

## 3. 滑动窗口策略

```
时间 ├───┬───┬───┬───┬───┬───┬───┬───┤
     KF1 KF2 KF3 KF4 KF5 KF6 KF7 KF8
         ↑←──── 滑动窗口 (3 KFs) ────→│
         固定   优化   优化
```

- **窗口大小**：3 个关键帧
- **固定帧**：窗口内最早的帧（保持全局参考）
- **运行频率**：每 10 个关键帧运行一次（平衡精度与速度）

## 4. 闭环检测

### 4.1 为什么需要闭环？

纯帧间跟踪会累积误差，导致轨迹"漂移"：

```
真实轨迹:    ○───────○───────○ (回到起点)
估计轨迹:    ○───────○───────○ (漂移到别处)
                                       ↑ 误差累积
```

闭环检测能识别相机回到之前访问过的位置，并修正累积漂移。

### 4.2 检测方法

当前使用**几何距离**检测：

```python
def detect_loop_closure(map, current_kf, all_kfs):
    for each distant keyframe:
        dist = distance(current_kf.position, kf.position)
        if dist < threshold:
            candidate = kf
    return candidate
```

更先进的方法（未来改进）：
- **DBoW2**：词袋模型，基于视觉词汇匹配
- **几何验证**：特征匹配 + 基础矩阵 / 本质矩阵
- **时序一致性**：连续多帧验证

### 4.3 闭环修正

当检测到闭环时，可以进行：
1. **位姿图优化**：将闭环约束加入图，优化所有关键帧位姿
2. **全局 BA**：用闭环约束重新优化所有位姿和点
3. **地图融合**：合并重复的地图点

## 6. 实验结果

| 指标 | 数值 |
|------|------|
| 总帧数 | 90 |
| 关键帧数 | 10 |
| 地图点数 | 12,613 |
| 轨迹总长 | 1.85 m |
| 有效跟踪率 | ~100%（所有帧均成功跟踪）|

### 轨迹分析

```
X (mm):     0 → -1700  (向左移动)
Y (mm):     0 → +200   (向上移动)  
Z (mm):     0 → -540   (向前移动)
```

相机运动趋势：向左前方移动并伴有俯仰角变化。

### 已知问题

1. **累积漂移**：BA 频率较低（每 10 KF），轨迹整体有漂移
2. **未验证闭环**：当前数据集中未触发闭环检测
3. **尺度漂移**：纯双目 SLAM 尺度由基线保证，但 PnP 仍可能引入微小尺度误差

## 7. 改进方向

| 方向 | 方法 | 预期效果 |
|------|------|----------|
| **更频繁的 BA** | 每帧运行局部 BA（优化最近 2-3 帧）| 减少局部漂移 |
| **全局 BA** | 检测闭环后运行全局 BA | 消除累积漂移 |
| **DBoW2** | 基于视觉词汇的闭环检测 | 更鲁棒的闭环识别 |
| **边缘化** | 舒尔补边缘化旧帧 | 保持计算量恒定 |
| **光度一致性** | 直接法对齐（如 DSO）| 提高弱纹理鲁棒性 |
| **IMU 融合** | 预积分 + VIO | 平滑轨迹，减少漂移 |

---

## 8. 总结

通过四个里程碑，我们从零实现了一个完整的双目 SLAM 系统：

| 里程碑 | 核心能力 | 关键算法 |
|--------|----------|----------|
| M1 | 立体匹配 + 深度图 | StereoSGBM, ORB |
| M2 | 帧间跟踪 + 位姿估计 | PnP + RANSAC |
| M3 | 地图构建 + 关键帧 | 三角化, 关键帧决策 |
| M4 | 后端优化 + 闭环 | 束调整 (BA), 闭环检测 |

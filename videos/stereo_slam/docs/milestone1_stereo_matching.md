# Milestone 1：立体匹配与深度图

## 1. 概述

本阶段实现了双目 SLAM 的数据加载和**立体匹配（Stereo Matching）** 模块，它是整个 SLAM 系统的**前端输入**——通过左右相机图像之间的视差计算每个像素的深度，为后续的运动估计和建图提供 3D 信息。

### SLAM 全景图

```
传感器输入 (左右图) ─→ 立体匹配 ─→ 深度图 ─→ 帧间跟踪 (Milestone 2)
                          │                      ↓
                          │                   位姿估计
                          │                      ↓
                          └──→ 三角化建图 (Milestone 3)
```

---

## 2. 核心原理：双目立体视觉

### 2.1 视差与深度

双目立体视觉模拟人眼通过**两个不同视角**观察同一场景，利用**三角测量**计算出深度。

**基本几何模型：**

```
       左相机                        右相机
        |                              |
        |                              |
        O_L ------------------------- O_R
              基线 B = 95mm
```

空间点 P 在左右相机图像上的投影点分别为 $p_L$ 和 $p_R$，它们在图像上的水平位置差称为**视差**：

$$d = x_L - x_R$$

根据相似三角形原理，深度 Z 与视差 d 的关系为：

$$Z = \frac{f \cdot B}{d}$$

其中：
- $f$：相机焦距（像素单位，即内参中的 $f_x$）
- $B$：双目基线（两个相机光心之间的距离，单位 mm）
- $d$：视差（像素单位）

### 2.2 为什么双目能直接测量深度？

| 方法 | 原理 | 特点 |
|------|------|------|
| **单目** | 从运动中恢复结构（SfM） | 深度有尺度模糊，无法直接获得真实尺度 |
| **双目** | 固定基线三角测量 | 直接获得真实尺度，无需运动 |
| **RGB-D** | 结构光/ToF | 直接测量深度，但易受环境光干扰 |

双目 SLAM 的核心优势是**直接获得真实尺度**，不需要像单目 SLAM 那样进行复杂的尺度初始化。

## 3. StereoSGBM 算法

### 3.1 算法选型

我们使用 OpenCV 的 **StereoSGBM**（Semi-Global Block Matching）算法，它是 Hirschmüller 2008 年提出的 SGM 算法的实现。

| 特性 | 说明 |
|------|------|
| **局部匹配** | 基于块（Block）的代价计算，使用 Census 变换 |
| **全局优化** | 沿多个方向（8 或 16 方向）的动态规划路径聚合 |
| **输出** | 每个像素的视差值（1/16 像素精度）|

### 3.2 算法流程

```
左右图像 ─→ 预处理 (CLAHE) ─→ 代价计算 ─→ 路径聚合 ─→ 视差优化 ─→ 视差图
                                    │
                          Census 变换 + SAD
```

**步骤详解：**

1. **预处理**：对红外图像应用 CLAHE（对比度受限自适应直方图均衡化），增强纹理对比度。

2. **代价计算**：对左图中每个像素，在右图中沿水平极线搜索，计算 Census 变换后的 Hamming 距离作为匹配代价。

3. **路径聚合**：沿多个方向（上、下、左、右、对角线）进行动态规划，最小化全局能量函数：
   $$E(D) = \sum_p C(p, D_p) + \sum_q P_1 \cdot T[|D_p - D_q| = 1] + \sum_q P_2 \cdot T[|D_p - D_q| > 1]$$
   
   其中：
   - $C(p, D_p)$：像素 p 在视差 $D_p$ 处的匹配代价
   - $P_1$：相邻像素视差变化 1 的惩罚（平滑小梯度变化）
   - $P_2$：相邻像素视差变化 > 1 的惩罚（保留边缘）

4. **后处理**：
   - **唯一性检查**：最优视差与次优视差的比率必须超过阈值
   - **斑点滤波**：去除视差图中孤立的小区域
   - **左右一致性**（可选）：检查左右视差图的一致性，剔除遮挡区域的错误匹配

### 3.3 参数说明

```python
StereoSGBM_create(
    minDisparity=0,              # 最小视差
    numDisparities=64,           # 视差搜索范围（必须被16整除）
    blockSize=5,                 # 匹配块大小（奇数）
    P1=8*3*blockSize²,          # 平滑惩罚1（小变化）
    P2=32*3*blockSize²,         # 平滑惩罚2（大变化/边缘）
    uniquenessRatio=10,          # 唯一性比率（%）
    speckleWindowSize=100,       # 斑点窗口大小
    speckleRange=2,              # 斑点内最大视差变化
    mode=2                       # MODE_HH（高质量模式）
)
```

## 4. 特征提取与立体匹配（ORB）

除了密集视差图，我们还提取了 **ORB 特征** 用于稀疏匹配，这是为后续帧间跟踪做的准备。

### 4.1 ORB 特征

ORB（Oriented FAST and Rotated BRIEF）是一种高效的局部特征，结合了：
- **FAST 角点检测**：快速检测角点位置
- **BRIEF 描述子**：二进制描述子，用汉明距离度量相似度

ORB 的优势：
- 免费开源（SIFT/SURF 有专利）
- 计算速度快，适合实时 SLAM
- 具有旋转不变性和尺度不变性（通过图像金字塔）

### 4.2 特征匹配

左右图之间的特征匹配使用 **暴力匹配 + 比率测试**：

1. 对左图的每个描述子，在右图中找到最近邻和次近邻
2. 如果最近邻距离 < 0.75 × 次近邻距离，则接受该匹配
3. 比率测试能有效剔除模糊匹配

```
stereo_slam/
├── config.py       # 相机标定参数、算法超参数

    ├── result_000.png   # 左图+右图+视差图+深度图的拼贴
    ├── keypoints_left.png   # 左图上的ORB特征点
    └── stereo_matches.png   # 左右图的特征匹配连线
```

### 5.1 `config.py` — 参数配置

```python
# 相机内参（按 640x400→848x480 缩放）
K_LEFT = [[596.25, 0,   424],
          [0,      540, 240],
          [0,      0,   1]]

# 双目基线
BASELINE_MM = 95.0

# 左右相机之间的旋转和平移（R≈I，纯X轴平移）
R_STEREO = I
T_STEREO = [-95, 0, 0]

# SGBM 参数
SGBM_NUM_DISPARITIES = 64
SGBM_BLOCK_SIZE = 5
```

### 5.2 `dataset.py` — 数据加载

关键函数 `load_stereo_pairs()`：

1. **扫描目录**：读取 IR_LEFT/ 和 IR_RIGHT/ 中所有 PNG 文件
2. **解析时间戳**：从文件名中提取 `d` 时间戳（设备时间戳）
3. **配对**：以 `d` 时间戳为键，取左右图的时间戳交集
4. **加载图像**：使用 `imdecode`（支持中文路径）
5. **返回**：排序后的 `(timestamp, left_img, right_img)` 列表

**文件命名规则：**
```
{序列号}_{帧号}_d{设备时间戳}_s{系统时间戳}_IR_{侧边}_Y8_{宽}x{高}.png
```

### 5.3 `stereo.py` — 核心算法

| 函数 | 功能 | 输出 |
|------|------|------|
| `preprocess()` | CLAHE 增强对比度 | 增强后的灰度图 |
| `create_stereo_matcher()` | 创建 SGBM 匹配器 | StereoSGBM 对象 |
| `compute_disparity()` | 计算密集视差图 | 浮点视差图 (H×W) |
| `disparity_to_depth()` | 视差→深度 | 深度图 (mm) |
| `extract_orb_features()` | ORB 特征提取 | 关键点 + 描述子 |
| `match_stereo_features()` | 左右图特征匹配 | DMatch 列表 |

## 6. 实验结果分析

### 6.1 定量结果

| 指标 | 数值 | 分析 |
|------|------|------|
| 配对数 | 90 | 94左+92右中交集90对 |
| 每帧特征 | 300~1200 | 随着场景纹理变化 |
| 立体匹配 | 116~328 | 足够用于位姿估计（需要至少6对） |
| 深度覆盖 | 57%~78% | 视差有效像素占比 |
| 深度均值 | 4~9m | 室内场景典型范围 |

### 6.2 数据趋势

- **帧1-30**：特征多 (~1000+)，深度均值 ~4m → 相机在纹理丰富的近处区域
- **帧60-90**：特征减少 (~400-600)，深度均值 ~8m → 相机向远处空旷区域移动

### 6.3 局限性

1. **内参精度**：使用参考值缩放，未真实标定，深度有系统偏差
2. **视差范围**：64 级对于室内近距离场景分辨率不足
3. **无畸变校正**：假设图像已校正
4. **中文字符路径**：OpenCV 不支持中文路径，需用 `imdecode` 绕过

## 7. 关键数学公式汇总

| 公式 | 说明 |
|------|------|
| $Z = f \cdot B / d$ | 视差转深度 |
| $d = x_L - x_R$ | 视差定义 |
| $P = K [R \mid t] P_w$ | 3D 点投影到图像 (pinhole model) |
| $E(D) = \sum C(p,D_p) + \sum P_1 \cdot T[\Delta D = 1] + \sum P_2 \cdot T[\Delta D > 1]$ | SGM 能量函数 |
| $K = \begin{bmatrix} f_x & 0 & c_x \\ 0 & f_y & c_y \\ 0 & 0 & 1 \end{bmatrix}$ | 相机内参矩阵 |

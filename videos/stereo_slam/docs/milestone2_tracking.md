# Milestone 2：帧间跟踪与位姿估计

## 1. 概述

本阶段实现了双目 SLAM 的**帧间跟踪**模块，通过匹配连续帧之间的特征点，利用**PnP（Perspective-n-Point）算法**估计相机的运动。

```
帧 N-1 (已知位姿 + 3D点)
    │
    ├─ ORB特征提取
    ├─ 与帧 N 特征匹配
    │
帧 N (当前帧)
    │
    ├─ ORB特征提取  
    ├─ 与帧 N-1 匹配 → 3D-2D 对应关系
    ├─ PnP + RANSAC → 当前帧位姿
    └─ 更新地图点
```

---

## 2. 核心原理：PnP 算法

### 2.1 问题定义

**PnP（Perspective-n-Point）** 是根据已知的 3D 世界点与 2D 图像点的对应关系，求解相机位姿的问题。

给定：
- $n$ 个已知的世界坐标点 $P_i = (X_i, Y_i, Z_i)$
- 它们在图像上的投影位置 $p_i = (u_i, v_i)$
- 相机内参 $K$

求解：
- 相机在世界坐标系中的位姿 $[R \mid t]$

### 2.2 投影方程

$$
s \begin{bmatrix} u \\ v \\ 1 \end{bmatrix} = K \begin{bmatrix} R \mid t \end{bmatrix}
\begin{bmatrix} X \\ Y \\ Z \\ 1 \end{bmatrix}
$$

展开为：
$$
u = f_x \frac{R_{11}X + R_{12}Y + R_{13}Z + t_x}{R_{31}X + R_{32}Y + R_{33}Z + t_z} + c_x
$$
$$
v = f_y \frac{R_{21}X + R_{22}Y + R_{23}Z + t_y}{R_{31}X + R_{32}Y + R_{33}Z + t_z} + c_y
$$

PnP 求解的就是 $R$（旋转矩阵，3 DoF）和 $t$（平移向量，3 DoF），共 **6 个自由度**。

### 2.3 求解方法

我们使用 OpenCV 的 `solvePnPRansac`，它结合了：

| 组件 | 作用 |
|------|------|
| **EPnP** | 高效的 $O(n)$ PnP 求解器（作为内循环） |
| **RANSAC** | 鲁棒地剔除误匹配（外点） |

**RANSAC 流程：**
1. 随机采样 4 组 3D-2D 对应点
2. 用 EPnP 计算初始位姿
3. 计算所有点的重投影误差
4. 统计内点（误差 < 阈值）
5. 重复 N 次（本例 $N=200$）
6. 选取内点最多的结果，用所有内点重新优化

### 2.4 为什么需要至少 3 对点？

PnP 的每个 3D-2D 对应提供两个约束方程（$u, v$），未知数 6 个，因此理论上至少需要 **3 对点**（6 个方程）。但实践中视野内噪声影响较大，通常要求 **6 对以上** 以保证解的质量。

## 3. 帧间特征匹配

### 3.1 匹配策略

```
当前帧左图特征 ←── 暴力匹配 ──→ 上一帧左图特征
                          │
                    比率测试 (0.75)
                          │
                    有效匹配 ──→ 查询 3D 点
                          │
                    构建 3D-2D 对应
```

### 3.2 数据流

```
上一帧:
  左图 keypoints ──→ 3D 世界坐标 (从立体匹配三角化)
       │
匹配索引 ──→ 当前帧 keypoints → 2D 坐标
       │
       └──→ 3D-2D 对 → PnP
```

**关键数据结构 `prev_stereo_pt_ids`：**
```python
{left_kp_index: map_point_id, ...}
```

这是一个从上一帧左图关键点索引到地图点 ID 的映射，实现从 2D 特征到 3D 世界点的 O(1) 查询。

## 4. 代码实现

### `tracker.py` — 核心跟踪类

| 方法 | 功能 |
|------|------|
| `extract(image)` | ORB 特征提取，返回关键点和描述子 |
| `match_ratio_test(d1, d2)` | 比率测试匹配，返回优秀匹配 |
| `estimate_pose_pnp(pts_3d, pts_2d)` | PnP + RANSAC 位姿估计 |
| `rvec_tvec_to_matrix(r, t)` | 旋转向量+平移 → 4×4 变换矩阵 |

### PnP 参数

```python
cv2.solvePnPRansac(
    pts_3d, pts_2d,
    K_LEFT, D_IR,
    iterationsCount=200,      # RANSAC 迭代次数
    reprojectionError=8.0,    # 内点重投影误差阈值 (像素)
    confidence=0.99,          # 置信度
    flags=SOLVEPNP_ITERATIVE  # 迭代优化
)
```

### 位姿表示

所有位姿使用 **4×4 齐次变换矩阵**：

$$
T_{w \to c} = \begin{bmatrix}
R_{3\times3} & t_{3\times1} \\
0_{1\times3} & 1
\end{bmatrix}
$$

将世界坐标系中的点转换到相机坐标系：
$$P_{cam} = T_{w \to c} \cdot P_{world}$$

## 5. 退化情况处理

| 问题 | 表现 | 处理 |
|------|------|------|
| 特征太少 | 匹配数 < 6 | 跳过该帧 |
| 误匹配多 | PnP 内点少 | RANSAC 自动剔除 |
| 快速运动 | 匹配数骤减 | 增加 RANSAC 迭代次数 |
| 纹理缺失 | 特征数 < 50 | 不跟踪，等待下一帧 |

---

## 6. 局限与改进

1. **仅帧间跟踪**：没有全局优化，误差会随时间累积（漂移）
2. **纯视觉里程计**：不是完整的 SLAM（尚未包含闭环）
3. **PnP 依赖前一帧**：跟踪丢失后无法恢复
4. **改进方向**：
   - 恒速运动模型预测初始位姿
   - 参考关键帧匹配（不依赖前一帧）
   - IMU 融合（VIO）

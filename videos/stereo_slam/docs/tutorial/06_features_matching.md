## 6.1 什么是图像特征 🔴

### 6.1.1 直观理解

```
图像特征 = 图像中"特别"的点

平坦区域        边缘          角点（特征）
░░░░░░░░       ░░██░░       ░░██░░
░░░░░░░░       ░░██░░       ░░██▒▒
░░░░░░░░       ░░██░░       ░░░░▒▒
  没特征        一个方向        两个方向
              有变化          都有变化
```

**特征点 = 角点**（两个方向都有变化的点）。

### 6.1.2 为什么 SLAM 需要特征

```
1. 可重复检测：换个角度还能找到同一个点
2. 位置精确：亚像素精度
3. 有描述子：可以匹配
4. 分布均匀：覆盖整个图像

SLAM 用特征点作为"地标"来定位
```

## 6.2 ORB 特征 🔴

### 6.2.1 组成

ORB = **Oriented FAST** + **Rotated BRIEF**

```
ORB 特征 = FAST 角点位置 + BRIEF 描述子 + 方向信息
```

### 6.2.2 FAST 角点检测

FAST 的思想极其简单：

```
检查一个像素 p 周围 16 个像素：
  如果有连续 N 个像素比 p 亮/暗一定阈值
  → p 就是角点
```

```python
# FAST 检测
fast = cv2.FastFeatureDetector_create(threshold=20)
kp = fast.detect(gray, None)
```

**速度极快**，比 SIFT 快 10 倍以上，所以 ORB-SLAM 选择它。

### 6.2.3 BRIEF 描述子

BRIEF 用**二进制串**描述一个特征点周围的样子：

```
在关键点周围随机选 256 对像素：
  如果第一比第二亮 → 该位为 1
  否则 → 该位为 0

得到 256 位的二进制描述子：
  [1, 0, 1, 1, 0, 0, 1, 0, ...]
```

### 6.2.4 💡 二进制的好处

```python
# 比较两个二进制描述子的"距离"
# 用汉明距离（不同的位数）而不是欧氏距离

def hamming_distance(desc1, desc2):
    """二进制描述子的相似度"""
    return bin(desc1 ^ desc2).count('1')  # XOR + 数1

# 汉明距离计算比欧氏距离快几十倍
# 这就是 ORB 快的秘密
```

### 6.2.5 图像金字塔（尺度不变性）

```
原始图像       │
  ↓ 缩放      │
半尺寸图像     │      不同尺度的特征
  ↓ 缩放      │         ██  ██
更小图像       │       ██  ██  ██
              │     ██  ██  ██  ██

每层检测特征 → 合并 → 在不同尺度都能找到
```

### 6.2.6 方向（旋转不变性）

ORB 给每个特征点计算一个**主方向**，匹配时把描述子旋转到这个方向——所以旋转后还能匹配。

## 6.3 特征匹配 🔴

### 6.3.1 暴力匹配

```python
bf = cv2.BFMatcher(cv2.NORM_HAMMING)

# 一对一匹配
matches = bf.match(desc1, desc2)

# 一对二匹配（推荐，因为可以做比率测试）
matches = bf.knnMatch(desc1, desc2, k=2)
```

### 6.3.2 比率测试

```python
good = []
for m, n in matches:
    if m.distance < 0.75 * n.distance:
        good.append(m)
```

**💡 原理：**

```
左图特征 A：
  在右图中找到的最匹配 → m（距离 10）
  第二匹配 → n（距离 50）
  10 < 0.75×50=37.5 → 明确匹配 → 保留

左图特征 B：
  最匹配 → m（距离 30）
  第二匹配 → n（距离 32）
  30 > 0.75×32=24 → 模糊匹配 → 剔除
```

### 6.3.3 对称性测试

```python
# 左→右 和 右→左 都要匹配，且互相确认
matches_lr = match(desc_left, desc_right)
matches_rl = match(desc_right, desc_left)

sym_matches = []
for m in matches_lr:
    if matches_rl[m.trainIdx].trainIdx == m.queryIdx:
        sym_matches.append(m)
```

## 6.4 如何判断匹配质量 🔴

### 6.4.1 可视化

看匹配连线图：

```
好匹配                         坏匹配
┌──────┐  ┌──────┐           ┌──────┐  ┌──────┐
│  \   │  │  /   │           │ \    │  │   /  │
│   \  │  │ /    │           │  \   │  │   /  │
│    \ │  │/     │           │   \  │  │  /   │
└──────┘  └──────┘           └──────┘  └──────┘
  平行一致                      杂乱交叉

几乎所有匹配线平行 → 匹配好
匹配线交叉 → 有误匹配
```

### 6.4.2 统计指标

```python
# 理想匹配的分布特征：
# 1. 不同匹配的位移向量应该相似
# 2. 匹配距离直方图应该集中在低值
# 3. 内点率应该 > 50%

# 看匹配距离分布
distances = [m.distance for m in good_matches]
plt.hist(distances, bins=20)
plt.xlabel('Match distance')
plt.ylabel('Count')
```

## 6.5 SIFT 与 ORB 对比 🟡

| 特性 | ORB | SIFT |
|------|-----|------|
| 速度 | ⭐⭐⭐ 非常快 | ⭐ 慢 |
| 精度 | ⭐⭐ 一般 | ⭐⭐⭐ 很高 |
| 专利 | 免费 | 有专利（需付费） |
| 描述子 | 二进制，256位 | 浮点，128维 |
| 距离度量 | 汉明距离（XOR） | 欧氏距离 |
| SLAM 使用 | ORB-SLAM | 离线重建 |

**SLAM 一般用 ORB**，因为实时性要求高。

---

## 6.6 特征匹配在 SLAM 中的三种用途

```
1. 帧间匹配（跟踪）
   当前帧特征 ↔ 上一帧特征
   → PnP 求当前帧位姿

2. 立体匹配（深度）
   左图特征 ↔ 右图特征
   → 三角化求 3D 点

3. 闭环匹配
   当前帧特征 ↔ 关键帧特征
   → 检测是否回到之前的位置
```

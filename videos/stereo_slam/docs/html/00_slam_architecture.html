五、stereo.py — 立体匹配

stereo_slam/stereo.py，共约 70 行。

功能：两个算法——密集匹配（StereoSGBM 算每像素视差 → 深度图）+ 稀疏匹配（ORB 特征提取 + 左右图匹配）。

5.1 CLAHE 预处理

在 preprocess(image) 中：
创建 CLAHE 对象 → clipLimit=2.0, tileGridSize=(8,8) → apply(image)

为什么：红外图像对比度低，CLAHE 能增强局部纹理，让特征提取和立体匹配更准。

5.2 StereoSGBM

半全局块匹配（Semi-Global Block Matching），Hirschmüller 2008 年提出。

核心思想：
  1. 对每个像素，在右图同一行上搜索，计算匹配代价（Census 变换 + Hamming 距离）
  2. 沿多个方向做动态规划，聚合代价
  3. 选择代价最小的视差值
  4. 后处理：唯一性检查、斑点滤波
关键参数：
numDisparities=64：搜索 64 个视差级别
blockSize=5：5×5 匹配块
P1=8×3×25, P2=32×3×25：平滑惩罚项
mode=2 (MODE_HH)：高质量模式

5.3 视差转深度

\[ Z = \frac{f_x \cdot B}{d} \] $f_x = 596.25$（焦距像素），$B = 95$ mm（基线），$d$ 为视差（像素）

5.4 ORB 特征提取

ORB = Oriented FAST + Rotated BRIEF

FAST：检测角点（像素周围 16 个像素中至少有 N 个比它亮/暗）
BRIEF：256 位二进制描述子（随机 256 对像素比较亮度）
方向：用灰度质心法计算主方向 → 旋转不变性
尺度：图像金字塔 → 尺度不变性

cv2.ORB_create(nfeatures=800, scaleFactor=1.2, nlevels=4)

5.5 左右图特征匹配

match_stereo_features(desc_left, desc_right, ratio_thresh=0.75)

① BFMatcher(NORM_HAMMING) 做 knnMatch(d1, d2, k=2)
② 对每对 (m, n)：m.distance < ratio × n.distance → 保留
③ 返回保留的 DMatch 列表

比率测试的意义：如果最近邻和次近邻太接近 → 匹配不明确 → 宁可不要。

6.2 PnP + RANSAC

输入：3D 点世界坐标 + 2D 像素坐标 + 相机内参 K
输出：旋转向量 rvec (3,) + 平移向量 tvec (3,)

cv2.solvePnPRansac(pts_3d, pts_2d, K, D_IR,
  iterationsCount=200, reprojectionError=8.0,
  confidence=0.99, flags=SOLVEPNP_ITERATIVE)
目标： $\displaystyle \min_{R,t} \sum_{i \in inliers} \left\| \pi(R,t; P_i) - p_i \right\|^2$

6.3 变换矩阵转换

7.2 三角化

\[ \text{cv2.triangulatePoints}(P_1, P_2, pts_l, pts_r) \] 其中 $P_1 = K[I \mid 0]$, $P_2 = K[R \mid t]$ 分别为左右相机的 3×4 投影矩阵

返回 4×N 齐次坐标 → 归一化为 3D 坐标：

\[ \mathbf{P} = \begin{bmatrix} X/W \\ Y/W \\ Z/W \end{bmatrix} \]

7.3 关键帧判断

def is_keyframe(prev_pose, curr_pose, tracked_ratio):
  delta = prev_pose⁻¹ @ curr_pose   # 相对变换
  trans = ||delta[:3, 3]||      # 平移距离
  angle = arccos((tr(R)-1)/2)    # 旋转角度
  return trans > 200mm or angle > 0.15rad or ratio < 0.4

八、optimizer.py — 束调整与闭环

stereo_slam/optimizer.py，共约 120 行。

8.1 局部 BA

优化变量：滑动窗口内 KF 的位姿 (6 DoF) + 观测到的 3D 点 (3 DoF)
目标函数：最小化所有观测的重投影误差

$\displaystyle \min_{\{R_i,t_i\},\{X_j\}} \sum_{(i,j)} \| \pi(R_i,t_i; X_j) - x_{ij} \|^2$
实现：scipy.optimize.least_squares
method='trf', max_nfev=10, ftol=1e-2, xtol=1e-2

每次取最近 n=BA_WINDOW（默认 3）个关键帧：
 ① 收集这些 KF 共同观测到的 3D 点
 ② 构建参数向量：先所有 KF 位姿（6×n），再所有 3D 点（3×m）
 ③ 构建残差函数：遍历每个观测，计算 (u_proj - u_obs, v_proj - v_obs)
 ④ 调用 least_squares 优化
 ⑤ 更新优化后的 KF 位姿和 3D 点坐标

8.2 闭环检测

两个关键帧的空间距离很近，但它们在时间上相隔很远 → 可能形成了闭环。

detect_loop_closure(map, current_kf, all_kfs)
 ① 遍历所有 KF（跳过最近 LC_SEARCH_DIST 个）
 ② 计算与当前 KF 的空间距离
 ③ 如果距离 < 2×KF_MIN_TRANSLATION → 候选闭环
 ④ 返回最近的候选

函数清单

函数输入输出

detect_loop_closure(map, cur_kf, all_kfs)地图+当前 KF闭环 KF or None

10.3 _process_frame() — 每帧处理流程（核心）

def _process_frame(self, idx, ts, left_img, right_img):

  # ① 增强
  left_enh = self.clahe.apply(left_img)
  right_enh = self.clahe.apply(right_img)

  # ② 特征提取
  kp_left, desc_left = self.tracker.extract(left_enh)
  kp_right, desc_right = self.tracker.extract(right_enh)

  # ③ 立体匹配（左右图）→ 三角化 → 3D 点
  stereo_matches = self.tracker.match_ratio_test(desc_left, desc_right)
  pts_3d_cam = triangulate_stereo(kp_left, kp_right, stereo_matches, ...)
  过滤：Z > 100mm

  # ④ 第一帧：初始化
  if idx == 0:
    pose = I          # 第一帧位姿设为单位矩阵
    3D点加到 Map
    存储 kp_idx ↔ point_id 的映射
    创建第一个 KF
    return

  # ⑤ 帧间匹配
  track_matches = match_ratio_test(desc_left, self.prev_desc)

  # ⑥ 构建 3D-2D 对应 → PnP
  遍历 track_matches:
    if trainIdx in prev_stereo_pt_ids:
      3D点 = map.points[prev_stereo_pt_ids[trainIdx]]
      2D点 = kp_left[queryIdx]
      加入 (3D, 2D) 列表
  rvec, tvec, inlier_mask = estimate_pose_pnp(pts_3d, pts_2d)
  pose = rvec_tvec_to_matrix(rvec, tvec)

  # ⑦ 新 3D 点 → 世界坐标 → 加入 Map
  pose_inv = inv(pose)
  pts_3d_world = (R_inv @ pts_3d_cam.T + t_inv).T
  添加到 map.points{}

  # ⑧ 关键帧判断
  if is_keyframe(prev_pose, pose, tracked_ratio):
    创建 KF → 加入 map.keyframes[]
    局部 BA
    闭环检测

  # ⑨ 更新状态
  self.prev_kp, prev_desc = kp_left, desc_left
  self.prev_pose = pose
  self.prev_stereo_pt_ids = merged_pt_ids
  self.all_poses.append(pose[:3, 3])  # 记录轨迹
